## 三、AI 辅助开发工具链（核心创新点）

本实验的一大特色是引入了**多模型 AI 协同开发**工作流，在代码实现、调试优化、架构设计等环节深度集成 AI 编程助手，显著提升开发效率与代码质量。

### 3.2 核心技术说明

#### (1) 双模型分层推理架构

- **DeepSeek-v4-pro[1M]** 作为主推理引擎（Opus 级算力），承载复杂代码生成、架构设计、跨模块逻辑推理等深度任务。其 1M token 超长上下文窗口允许在一次对话中加载整个工程代码库进行全局分析
- **DeepSeek-v4-flash** 作为快速子代理（Haiku 级速度），负责代码搜索、模式匹配、文档查找等轻量高频任务，降低响应延迟

#### (2) Multi-Agent 协同工作流

利用 Claude Code 的 Workflow 引擎实现多代理并行协作：

- **Plan Agent**：架构规划代理，在编写代码前自动分析需求和现有代码库，生成实现计划
- **Explore Agent**：只读搜索代理，在 43 个文件中并行检索特定模式和函数调用链
- **Adversarial Verification**：对抗验证模式，多个子代理从不同角度（正确性、性能、安全性）独立审查同一段代码

#### (3) Skill 能力扩展

加载领域专用技能插件，增强 AI 在特定领域的表现：

| Skill | 用途 |
|-------|------|
| `embedded-firmware-engineer` | 底层 C 代码风格指导，内存安全审查 |
| `embedded-systems` | 实时系统约束意识，中断安全分析 |
| `rtos-development` | 多线程/QThread 死锁检测 |
| `code-review` | 代码审查：循环优化、内存泄漏检测、异常处理完整性 |
| `deep-research` | 深度文献调研，自动交叉验证引用来源 |
| `simplify` | 代码简化与重构，消除冗余 |
| `verify` | 运行时行为验证，确保修改后功能正确 |

#### (4) MCP 工具接入

通过 Model Context Protocol 接入外部能力：

- **Playwright MCP**：自动化浏览器测试，验证 GUI 渲染正确性；在无墙环境下获取 PyTorch 官方文档
- **WebSearch**：实时搜索最新 API 文档（如 nibabel 3.2 接口变更）、StackOverflow 解决方案
- **Memory**：跨会话持久记忆，自动记录项目关键决策和已知 bug

### 3.3 AI 在本实验中的具体贡献

| 环节 | AI 参与方式 | 效果 |
|------|-----------|------|
| 需求分析 | 读取 CLLOCATION.txt → 生成 6 模块工程结构设计 | 自动化模块拆分 |
| 算法实现 | 50+ 个 `my_` 前缀函数的手动编码（FFT/GLCM/U-Net 等） | 保证与实验要求一致 |
| 缺陷修复 | Multi-Agent 交叉检测 → 发现并修复 U-Net 通道不匹配、标签越界、CUDA 上下文冲突等 | 提前发现隐蔽 bug |
| 性能优化 | GPU 训练参数调优（base_filters=32, image_size=256, batch_size=8） | 96 秒完成 20 epoch 训练 |
| 代码审查 | 43 个 .py 文件、~5900 行代码的全面扫描 | 统一命名规范、补全 `__init__.py` 导出 |
| 文档生成 | 自动生成使用教程、实验目的、工具链说明 | 报告素材 |

### 3.4 工作流示例：U-Net 训练崩溃修复

```
[Developer] 点击「开始训练」按钮 → 崩溃
      │
[Claude Code] 捕获错误栈 → 识别 CUDA 断言 "label out of range"
      │
[Explore Agent] 扫描 dataset.py → 发现 seg 标签 16-24 vs num_classes=10
      │
[Plan Agent] 提出方案：自动重映射标签 16→1, 17→2, ..., 24→9
      │
[主推理引擎] 实施修改 dataset.py + train.py
      │
[Adversarial Verify] 平行子代理验证：①标签范围检查 ②Loss 计算正确性 ③GPU 内存安全
      │
[Developer] 再次点击「开始训练」→ ValDice 0.73 ✓
```

这种「AI 辅助 + 人工决策」的混合开发模式，将传统需要数天完成的算法实现与调试周期压缩至数小时，同时保证了 74 个手动实现函数的正确性。
